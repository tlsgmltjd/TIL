# cookie session token

### HTTP 통신은

`요청(Request)` -> `응답(Response)` 이 종료되면 연결을 끊는 처리 방식이다.

로그인과 같은 일을 할 때, `'누가' `로그인 중인지 상태를 `기억`하기 위해 **쿠키, 세션, 토큰**을 사용한다.

---

## 🍪 쿠키

- 쿠키는 공개 해도 되는 정보를 사용자의 브라우저에 저장시킨다.
- 브라우저에 저장되는 작은 텍스트 조각

## 세션

- 세션은 비밀번호와 같은 인증 정보를 쿠키에 저장하지 않고 대신에 사용자의 식별자인 `세션 id` 를 저장한다.
- 서버에는 인증 정보와 더불어 이 ID에 해당하는 사용자의 정보를 저장한다.

## 토큰 - JWT (JSON Web Token)

- 사용자가 로그인을 요청 시 서버에서는 어떠한 데이터를 기반으로 인증이 가능한 `토큰`을 만들어서 `클라이언트에 전달`하는 방식

**Authentication** : 인증. 유저가 누구인지 확인하는 절차 `로그인` `회원가입`  
**Authorization** : 인가. 유저에 대한 권한을 허락하는 것.
